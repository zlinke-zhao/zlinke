## 引子：Agent 从云端坠落到你的桌面

过去两年，AI Agent 一直悬在云端。你在网页上跟它聊天，它在服务器上运行，数据要来回传，文件要手动上传下载。Agent 很聪明，但始终隔着一层"云"。

## 二、技术硬核：K2.6 + Agent 集群 + WebBridge

Kimi Work 的"肌肉"来自三块核心技术。

### 2.1 Kimi K2.6：不止是个模型

K2.6 是月之暗面 4月开源的旗舰模型，1T 参数 MoE 架构（32B 激活参数），256K 上下文。但在 Kimi Work 里，它不只是一个"更聪明的大模型"。

| 能力 | 数据 | 说明 |
|---|---|---|
| **不间断编码** | 13 小时连续 | 写入/修改 4000+ 行代码 |
| **工具调用** | 4000+ 次自主调用 | 浏览器、API、本地命令 |
| **Agent 集群** | 300 个子 Agent 并行 | 自主拆解 + 调度 + 收拢结果 |
| **多模态输入** | 原生支持图片+视频 | 不依赖外挂 OCR/转写 |

> 关键区别：K2.6 在设计时就以 "Agent 编排" 为目标，不是事后拼凑的 tool-calling 方案。这也是为什么它能稳定支撑 300 个 Agent 同时跑——底层架构就是为多智能体设计的。

### 2.2 Agent 集群（Agent Swarm）：300 个 AI 同时为你干活

这是 Kimi Work 最与众不同的能力。

传统 Agent 是一个"模型 + 工具"的串行流程：想一步、做一步、看结果、再想一步。Kimi Work 的 Agent 集群打破了这个模式：

```
用户：分析2026年新能源汽车市场，生成150页投研报告

Kimi Work 自动拆解：
├─ Agent 1~10：数据采集组
│  ├─ Agent 1：同花顺抓取A股新能源板块数据
│  ├─ Agent 2：Yahoo Finance 抓美股数据
│  ├─ Agent 3：天猫/京东抓取销量评论
│  └─ Agent 4~10：行业网站/政策文件爬取
│
├─ Agent 11~20：数据分析组
│  ├─ Agent 11：比亚迪财报分析
│  ├─ Agent 12：特斯拉财报对比
│  └─ Agent 13~20：各细分赛道数据清洗
│
├─ Agent 21~25：报告撰写组
│  ├─ Agent 21：行业总览章节
│  ├─ Agent 22：竞争格局章节
│  └─ Agent 23~25：投资建议章节
│
└─ Agent 26~30：质量审核组
   ├─ Agent 26：数据准确性校验
   ├─ Agent 27：格式排版
   └─ Agent 28~30：PPT 生成
```

这 300 个 Agent 不是噱头——在真实场景中，一个复杂研报确实需要数据采集、清洗、分析、撰写、审核、设计这六个环节并行推进。

### 2.3 WebBridge：让 AI 真的会"上网"

这是 Kimi Work 的另一个独特能力。大部分 AI 工具只能通过 API 访问网页，但 Kimi Work 的 WebBridge 可以像人类一样操控浏览器：

- **点击、滚动、翻页**：像真人一样浏览动态加载页面
- **抓取深层数据**：包括公众号后台、同花顺等需要渲染才能看到的内容
- **自动填表**：在网页上填写表单并提交

据社区实测，Claude Code 的 Chrome 插件在处理公众号后台这类动态渲染页面时会直接报错，而 Kimi Work 的 WebBridge 可以正常处理。这个差距对需要处理中文互联网数据的用户来说，是实质性的。

## 三、三步上手：从下载到第一个任务

### Step 1：下载安装

### Step 2：切换 Work 模式

安装完成后打开 Kimi 客户端，你会看到熟悉的聊天界面。在左侧导航栏，从 **「Chat」** 切换到 **「Work」** 标签页——界面立刻从"聊天框"变成"项目管理面板"。

### Step 3：创建你的第一个任务

点击「新建任务」，用自然语言描述目标。例如：

```
帮我调研过去一周 AI 领域的重大新闻，
整理成一份 10 页的市场简报 PPT。
注意：重点关注国内大模型和 Agent 方向。
```

Kimi Work 会自动：
1. 拆解任务（搜新闻 → 筛选重点 → 整理分析 → 生成PPT）
2. 通过 WebBridge 访问新闻网站
3. 调用内置 Skill 生成 PPT
4. 将结果保存到本地文件夹

## 四、核心功能深度拆解

### 4.1 本地文件处理：你的私人"文件夹管家"

Kimi Work 可以直接挂载本地文件夹，读取、创建、整理文件。不是上传到云端再处理——是真·本地操作。

实用场景：
- **文件夹整理**："帮我把这个目录下所有 PDF 按主题分类到子文件夹"
- **批量处理**："这 47 份合同 PDF 中，提取甲乙双方名称和金额，汇总到 Excel"
- **文档生成**："基于这个文件夹里的实验数据，生成一份学术论文初稿"

安全方面，Kimi Work 在执行任何文件操作前都会弹出系统确认窗口（Ask before acting），不会偷偷删你的文件。

### 4.2 Skill 系统：可扩展的能力插件

Skill 是 Kimi Work 的"技能包"——预置的专业能力模块：

| Skill | 功能 |
|---|---|
| 建站 Skill | 一句话生成可部署的网站 |
| PPT Skill | 将文字/数据转化为演示文稿 |
| 金融分析 Skill | 调用同花顺/天眼查数据生成研报 |
| 数据分析 Skill | 清洗数据、画图表、写分析 |
| 自定义 Skill | 用户可以创建自己的 Skill |

这和 ChatGPT 的 GPTs 类似，但 Kimi Work 的 Skill 能直接调用本地资源（文件、浏览器、专业数据库），而不是被困在沙箱里。

### 4.3 专业数据库：金融从业者的杀手锏

Kimi Work 内置接入：
- **A 股/港股**：同花顺
- **企业工商**：天眼查
- **机构数据**：iFind
- **美股/全球**：Yahoo Finance

无需繁琐的 API 配置，在对话中直接引用。对于金融从业者，这意味着："帮我对比宁德时代和比亚迪近三年的毛利率变化趋势"——Kimi Work 可以直接从数据库拉数据、画图、写分析，不用你手动找年报。

### 4.4 定时任务：7×24 自动化流水线

Kimi Work 支持 Cron 定时任务，可以设置：
- **每日晨报**："每天早上 8 点，抓取 AI 领域最新新闻，生成简报发到桌面"
- **数据监控**："每 30 分钟检查一次股价，触发条件时弹窗提醒"
- **自动备份**："每周日晚 12 点，压缩项目文件夹备份到指定目录"

电脑休眠了怎么办？Kimi Work 提供"保持唤醒"选项，确保夜间任务如期执行。

## 七、优势与槽点：真实盘点

### ✅ 五大优势

1. **零门槛**：GUI 图形界面，不用终端、不用命令、不用配置。金融分析师和科研人员不需要会写代码
2. **Agent 集群独一份**：300 子 Agent 并行——其他桌面 Agent 产品目前没有这个能力
3. **WebBridge 是杀手锏**：能操控浏览器获取动态渲染数据（公众号后台、同花顺等），这是 API 访问做不到的
4. **专业数据生态**：同花顺/天眼查/iFind 等金融数据源原生接入，金融从业者刚需
5. **本地安全**：数据不出设备，文件操作有确认弹窗

### ⚠️ 四个槽点

1. **Agent 集群门槛高**：最核心的功能要 ¥199/月起，免费版只能"尝尝味道"
2. **Beta 阶段稳定性待验证**：社区反馈中，存在终端命令执行失败、跨平台兼容性问题
3. **macOS 限制**：只支持 Apple 芯片（M 系列），Intel Mac 被排除在外
4. **92% AI 生成代码**：听起来很酷，但也意味着测试覆盖可能不如传统工程模式——Bug 概率更高
